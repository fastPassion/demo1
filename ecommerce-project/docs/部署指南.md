# 部署指南

## 环境要求

- Node.js 18+
- MySQL 8.0+
- Redis (可选，用于缓存)
- Nginx (推荐用于生产环境)

## 开发环境设置

### 1. 克隆项目

```bash
git clone <repository-url>
cd ecommerce-project
```

### 2. 前端设置

```bash
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm start

# 构建生产版本
npm run build
```

### 3. 后端设置

```bash
cd backend

# 安装依赖
npm install

# 创建环境变量文件
cp .env.example .env

# 编辑 .env 文件配置环境变量
nano .env

# 启动开发服务器
npm run dev

# 启动生产服务器
npm start
```

### 4. 环境变量配置

创建 `.env` 文件：

```env
# 服务器配置
PORT=5000
NODE_ENV=development

# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_NAME=ecommerce
DB_USER=root
DB_PASSWORD=123456
DB_DIALECT=mysql

# JWT配置
JWT_SECRET=your-super-secret-jwt-key-change-in-production
JWT_EXPIRE=30d

# CORS配置
ALLOWED_ORIGINS=http://localhost:3000

# 文件上传配置
MAX_FILE_SIZE=5242880
UPLOAD_PATH=uploads
```

## 生产环境部署

### 1. 前端部署

```bash
# 在frontend目录中
npm run build
```

将 `frontend/build` 目录部署到：
- Netlify
- Vercel
- AWS S3 + CloudFront
- Nginx 静态文件服务

### 2. 后端部署

#### 使用 PM2 (推荐)

```bash
# 全局安装 PM2
npm install -g pm2

# 启动应用
pm2 start server.js --name "ecommerce-api"

# 开机自启动
pm2 startup
pm2 save
```

#### 使用 Docker

创建 `Dockerfile`：

```dockerfile
FROM node:18-alpine

WORKDIR /app

COPY package*.json ./
RUN npm ci --only=production

COPY . .

EXPOSE 5000

CMD ["node", "server.js"]
```

创建 `docker-compose.yml`：

```yaml
version: '3.8'
services:
  mongodb:
    image: mongo:6
    restart: always
    ports:
      - "27017:27017"
    volumes:
      - mongodb_data:/data/db
    environment:
      MONGO_INITDB_ROOT_USERNAME: root
      MONGO_INITDB_ROOT_PASSWORD: example

  backend:
    build: ./backend
    restart: always
    ports:
      - "5000:5000"
    environment:
      - NODE_ENV=production
      - MONGODB_URI=mongodb://mongodb:27017/ecommerce
      - JWT_SECRET=your-production-jwt-secret
      - ALLOWED_ORIGINS=https://yourdomain.com
    depends_on:
      - mongodb

  frontend:
    build: ./frontend
    restart: always
    ports:
      - "80:80"
    depends_on:
      - backend

volumes:
  mongodb_data:
```

启动服务：

```bash
docker-compose up -d
```

### 3. Nginx 配置

创建 Nginx 配置文件 `/etc/nginx/sites-available/ecommerce`：

```nginx
# 前端配置
server {
    listen 80;
    server_name yourdomain.com;

    location / {
        root /var/www/ecommerce-frontend/build;
        try_files $uri /index.html;
    }

    # API 代理
    location /api/ {
        proxy_pass http://localhost:5000/api/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }

    # 静态文件上传
    location /uploads/ {
        alias /var/www/ecommerce-backend/uploads/;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}

# HTTPS 配置（推荐）
server {
    listen 443 ssl http2;
    server_name yourdomain.com;

    ssl_certificate /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;

    # SSL 配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES256-GCM-SHA512:DHE-RSA-AES256-GCM-SHA512:ECDHE-RSA-AES256-GCM-SHA384:DHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers off;

    # 前端配置
    location / {
        root /var/www/ecommerce-frontend/build;
        try_files $uri /index.html;
    }

    # API 代理
    location /api/ {
        proxy_pass http://localhost:5000/api/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }

    # 静态文件上传
    location /uploads/ {
        alias /var/www/ecommerce-backend/uploads/;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/ecommerce /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## 数据库备份

### MySQL 备份脚本

创建备份脚本 `backup-mysql.sh`：

```bash
#!/bin/bash

# 配置
BACKUP_DIR="/backup/mysql"
DATE=$(date +%Y%m%d_%H%M%S)
DB_HOST="localhost"
DB_USER="root"
DB_PASSWORD="123456"
DB_NAME="ecommerce"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 执行备份
mysqldump -h $DB_HOST -u $DB_USER -p$DB_PASSWORD $DB_NAME > "$BACKUP_DIR/backup_$DATE.sql"

# 压缩备份
tar -czf "$BACKUP_DIR/backup_$DATE.tar.gz" "$BACKUP_DIR/backup_$DATE.sql"

# 删除未压缩的备份
rm -f "$BACKUP_DIR/backup_$DATE.sql"

# 删除7天前的备份
find $BACKUP_DIR -name "backup_*.tar.gz" -mtime +7 -delete

echo "MySQL备份完成: $BACKUP_DIR/backup_$DATE.tar.gz"
```

添加定时任务：

```bash
# 每天凌晨2点执行备份
crontab -e
0 2 * * * /path/to/backup-mongodb.sh
```

## 故障排除

### 常见问题

1. **MongoDB连接失败**
   - 检查MongoDB服务是否运行
   - 验证连接字符串
   - 检查防火墙设置

2. **端口被占用**
   - 使用 `netstat -tulpn | grep :5000` 查找占用端口的进程
   - 使用 `kill -9 <PID>` 终止进程

3. **内存不足**
   - 增加服务器内存
   - 优化Node.js内存使用
   - 使用PM2集群模式

4. **文件上传失败**
   - 检查上传目录权限
   - 验证文件大小限制
   - 检查磁盘空间
